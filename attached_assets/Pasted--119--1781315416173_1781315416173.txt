서울시 전역을 대상으로 하는 「서울 119 화재위험 예보 앱」을 만들어줘.

이 앱은 과거 화재통계, 서울시 화재발생 현황, 소방안전 빅데이터 플랫폼 자료, 소방청 화재발생 정보, 소방청 연간화재통계, 서울시 소방 화재 출동 현황, 그리고 기상청 예보자료를 활용하여 서울의 계절별·시간대별·지역별 화재위험도를 예측하고 시민과 소방관서에 맞춤형 예방 알림을 제공하는 서비스이다.

앱 이름은 「서울 119 화재위험 예보」로 한다.

지도는 네이버 지도를 사용하지 않고, 다음지도 즉 카카오맵만 사용한다.

핵심 목적은 다음과 같다.

1. 서울시 과거 화재 발생 데이터를 분석하여 월별, 계절별, 요일별, 시간대별, 지역별 화재 패턴을 파악한다.
2. 기상청 예보자료를 활용하여 오늘과 내일의 기상 위험요인을 반영한다.
3. 기온, 최고기온, 최저기온, 습도, 강수확률, 강수량, 풍속, 건조 여부, 폭염 가능성, 한파 가능성 등을 화재위험도 산정에 반영한다.
4. 서울시 자치구·행정동별 지역 특성인 노후건축물, 전통시장, 음식점 밀집지역, 산림 인접지역, 공동주택, 취약계층 밀집지역 등을 반영한다.
5. 오늘의 화재위험도를 관심, 주의, 경계, 위험 4단계로 산정한다.
6. 시민에게는 맞춤형 화재예방 알림을 제공하고, 소방관서에는 예방순찰·점검 우선지역을 추천한다.
7. 카카오맵을 통해 서울 전역의 화재위험도를 지도 기반으로 직관적으로 확인할 수 있게 한다.

카카오맵 API 키는 코드에 직접 넣지 말고 환경변수로 관리한다.

예:
- KAKAO_MAP_API_KEY
- KMA_API_KEY

화재위험도 점수는 100점 만점으로 산정한다.

위험도 산정 기준:
- 최근 5년간 같은 월·계절·시간대 화재 발생 빈도: 30점
- 해당 지역의 과거 화재 발생 빈도: 20점
- 발화요인 반복성: 15점
- 기상청 예보 기반 오늘의 기상 위험도: 20점
- 노후건축물, 전통시장, 음식점, 산림 인접 등 지역 위험요인: 15점

기상 위험도 세부 기준:
- 고온·폭염 예보 시 냉방기기, 실외기, 전기화재 위험 가중
- 한파 예보 시 전기장판, 난방기기, 멀티탭 과부하 화재 위험 가중
- 건조한 날씨와 낮은 습도 예보 시 산림 인접지역, 담배꽁초, 야외소각 화재 위험 가중
- 강풍 예보 시 외부 화재 확산 위험 가중
- 강수량이 적고 건조한 기간이 지속될 경우 임야·쓰레기·야외화재 위험 가중
- 습도가 높고 강수확률이 높은 경우 야외화재 위험은 낮추되, 실내 전기화재 위험은 별도 유지

점수별 위험등급:
- 0~30점: 관심
- 31~60점: 주의
- 61~80점: 경계
- 81~100점: 위험

앱 주요 화면은 다음과 같이 구성한다.

1. 메인 화면
- 오늘의 서울 전체 화재위험도 표시
- 현재 위험등급 표시
- 가장 위험한 화재유형 표시
- 기상청 예보 기반 주요 위험요인 표시
- 오늘의 주요 예방 메시지 표시
- 카카오맵 기반 위험지도 바로가기 버튼 표시

예시 문구:
“오늘 밤 서울 주택가 전기난방기 화재위험이 높습니다. 기상청 예보상 최저기온이 낮아 난방기기 사용 증가가 예상됩니다. 취침 전 전기장판과 멀티탭 전원을 확인하세요.”

2. 카카오맵 지도 화면
- 카카오맵 API를 활용하여 서울시 지도를 표시한다.
- 서울시 25개 자치구별 화재위험도를 색상 오버레이로 표시한다.
- 행정동 단위까지 확대하면 행정동별 위험도를 표시한다.
- 관심은 초록색, 주의는 노란색, 경계는 주황색, 위험은 빨간색으로 표시한다.
- 위험등급이 높은 지역에는 마커를 표시한다.
- 사용자가 자치구 또는 행정동을 클릭하면 상세 팝업을 표시한다.

상세 팝업 내용:
- 지역명
- 오늘의 화재위험 점수
- 위험등급
- 주요 화재위험 유형
- 위험 시간대
- 기상청 예보 기반 위험요인
- 지역 특성 위험요인
- 시민 예방수칙
- 소방관서 추천 조치

3. 지역 상세 화면
- 선택한 자치구 또는 행정동의 오늘 화재위험도
- 최근 5년간 월별 화재 추이 그래프
- 시간대별 화재 발생 그래프
- 주요 발화요인 순위
- 주요 화재 장소 유형
- 오늘과 내일의 기상청 예보 요약
- 기상요인이 화재위험도에 반영된 이유
- 맞춤형 예방 메시지
- 카카오맵에서 해당 지역 위치 표시

4. 시민 알림 화면
- 계절·시간대·지역별 맞춤형 화재예방 알림 제공
- 기상청 예보를 반영하여 알림 메시지를 자동 생성한다.

알림 예시:
- 겨울철 한파 예보: 전기장판, 난방기기, 멀티탭 과부하 주의
- 여름철 폭염 예보: 에어컨, 실외기, 전기화재 주의
- 봄철 건조·강풍 예보: 산림 인접지역, 담배꽁초, 야외소각 주의
- 명절 전후: 음식물 조리, 가스레인지, 튀김유 과열 주의
- 야간 시간대: 취침 전 전열기구, 가스밸브 확인

5. 소방관서 대시보드 화면
- 오늘의 관할지역 화재위험 순위
- 위험등급이 높은 행정동 목록
- 기상청 예보 기반 위험유형 분석
- 위험유형별 예방활동 추천
- 예방순찰 추천지역
- 전통시장, 노후주택, 음식점 밀집지역 등 점검 우선순위 표시
- 카카오맵에서 위험지역 확인 가능

관서용 조치 예시:
“성동구 ○○동 / 위험등급 경계 / 주택 전기화재 위험 / 위험시간 21:00~02:00 / 한파 예보로 난방기기 사용 증가 예상 / 공동주택 안내방송 및 취약가구 안전확인 권고”

6. 통계 분석 화면
- 월별 화재 발생 추이
- 계절별 화재유형 분석
- 시간대별 화재 발생 비율
- 발화요인별 비율
- 지역별 화재위험 순위
- 기상청 예보와 화재위험도 관계 표시
- 기온, 습도, 강수확률, 풍속 변화에 따른 위험도 변화 그래프 표시

앱 디자인은 공공안전 서비스 느낌으로 만든다.
전체적인 분위기는 신뢰감 있고 직관적인 디자인으로 한다.
색상은 흰색 배경을 기본으로 하고, 위험도는 초록·노랑·주황·빨강 단계로 명확히 표현한다.
고령자도 쉽게 볼 수 있도록 글자는 크게 하고 버튼은 단순하게 만든다.
카카오맵 지도 화면은 복잡하지 않게 구성하고, 위험지역과 예방 메시지가 한눈에 보이게 한다.

사용자 유형은 두 가지로 나눈다.

시민용:
- 우리 동네 오늘의 화재위험도 확인
- 카카오맵으로 우리 동네 위험도 확인
- 기상청 예보 기반 맞춤형 예방수칙 확인
- 위험 알림 수신
- 계절별 화재예방 정보 확인
- 관심지역 등록

소방관서용:
- 관할지역 위험도 확인
- 카카오맵 기반 위험지역 확인
- 예방순찰 우선순위 확인
- 특정 시간대 위험지역 확인
- 점검 및 홍보 대상 추천
- 위험유형별 예방활동 추천
- 기상청 예보 기반 예방활동 계획 수립

데이터가 실제로 연결되지 않은 시제품 단계에서는 샘플 데이터를 사용한다.
샘플 데이터는 서울시 25개 자치구를 기준으로 생성하고, 각 구마다 위험점수, 위험등급, 주요 화재유형, 위험시간대, 예방메시지, 기상요인, 중심 좌표를 포함한다.

- 소방관서용 예방활동 추천
- 통계 그래프 시각화
- CSV 데이터 업로드 기능
- 향후 기상청 API와 공공데이터 API 연동이 가능한 구조

개발 구조는 다음과 같이 설계한다.

프론트엔드:
- React 또는 Next.js 기반 웹앱
- 모바일에서도 보기 좋은 반응형 디자인
- 지도는 카카오맵 API만 사용
- 그래프는 Chart.js 또는 Recharts 사용

지도 컴포넌트 구조:
- KakaoMapView
- RiskOverlay
- RiskMarker
- RegionPopup
- DistrictSearch
- FavoriteRegionSelector

백엔드:
- Node.js, Python FastAPI 또는 Firebase 중 하나로 구성
- 화재 데이터와 기상청 예보 데이터를 저장하고 위험점수를 계산
- 지도 화면에 필요한 자치구·행정동별 위험도 데이터를 API로 제공
- 기상청 예보 API와 연동 가능한 구조로 설계

API 예시:
- GET /risk/today
- GET /risk/districts
- GET /risk/district/{districtName}
- GET /risk/neighborhood/{neighborhoodName}
- GET /weather/forecast
- GET /alerts/today
- GET /map/districts
- GET /map/neighborhoods
- POST /data/upload

데이터베이스:
- FireIncident 테이블
- WeatherForecast 테이블
- DistrictRisk 테이블
- NeighborhoodRisk 테이블
- AlertMessage 테이블
- RegionProfile 테이블
- MapRegion 테이블

개인정보 보호 원칙:
- 신고자, 환자, 전화번호, 상세주소 등 개인정보는 사용하지 않는다.
- 위치 정보는 자치구 또는 행정동 단위로 집계한다.
- 내부 출동자료를 사용할 경우 반드시 비식별화된 통계자료만 사용한다.
- 시민 알림은 개인 위치를 정밀 추적하지 않고, 사용자가 선택한 관심지역 기준으로 제공한다.

최종 결과물은 공모전 시제품처럼 보이게 만들어줘.
메인 화면, 카카오맵 지도 화면, 지역 상세 화면, 시민 알림 화면, 소방관서 대시보드 화면, 통계 분석 화면을 포함한 완성도 높은 프로토타입으로 구성해줘.
